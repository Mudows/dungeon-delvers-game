### Mudanças do Período

**IA dos Inimigos**

Implementado o comportamento completo de perseguição. Cada inimigo executa por turno: ataque se adjacente, perseguição se o jogador estiver no raio de detecção com linha de visão livre, ou inatividade. A linha de visão é verificada pelo algoritmo de Bresenham — inimigos atrás de paredes não ativam mesmo dentro do raio 5×5. O pathfinding usa BFS, garantindo o caminho mais curto navegável e respeitando tiles ocupados por outros inimigos.

**Névoa de Guerra**

Confirmada como implementação via layer Darkness do Construct 3: fundo preto sólido com sprite `player_light` em blend *destination-out*, gerando uma máscara radial ao redor do jogador. Não há lógica de FoW nos scripts — a layer cuida disso inteiramente. A implementação em `map.js` (`updateVisibility`, `renderFog`) foi mantida como código inativo, disponível caso a abordagem mude no futuro. GDD atualizado para refletir a abordagem real (radial, não quadrado 5×5).

**Refatoração Arquitetural**

- `grid.js` removeu a dependência de `combat.js`. O método `move()` passa a aceitar um callback `onBumpAttack(actor, target)`, delegando a resolução de combate ao chamador (`main.js`). Grid agora lida exclusivamente com movimento e colisão.
- `loadFloor()` destrói sprites e barras de HP dos inimigos do andar anterior antes de limpar o array, evitando vazamento de instâncias no C3.
- `_floorTilesInRoom` em `map.js` corrigido para usar `isFloor()` em vez de comparar com `=== 0`, incluindo tiles de corredor como pontos válidos de spawn.
- `utils.js` criado para centralizar `randomInt`, que estava duplicada em `main.js` e `map.js`.

**Refatoração do Sistema de Combate**

O sistema anterior garantia 100% de acerto, tornando o desfecho determinístico e a derrota do jogador inevitável com o tempo. O novo sistema usa rolagem contestada:

```
attackRoll  = rand(0–10) + ATQ do atacante
defenseRoll = rand(0–10) + DEF do defensor

acerto  → attackRoll > defenseRoll  → dano = ATQ_arma + (attackRoll − defenseRoll)
erro    → defenseRoll ≥ attackRoll  → sem dano
```

A fórmula se aplica igualmente a ataques do jogador e de inimigos. O dano mínimo ao acertar é 1 (sem arma equipada). A amplitude de 0–10 em cada rolagem mantém os stats relevantes sem tornar o resultado previsível. Calibragem dos valores virá com testes.

### Arquitetura Atual

- `grid.js` — coordenadas, movimentação, detecção de colisão. Sem dependências de combate.
- `map.js` — geração procedural via BSP com MST para conectividade garantida.
- `themes.js` — configuração visual de tiles por fase com sorteio ponderado.
- `turn.js` — ciclo de turnos, factory de inimigos, classe Enemy com IA (BFS + Bresenham LOS).
- `combat.js` — resolução de combate físico por rolagem contestada.
- `hud.js` — interface do jogador integrada ao editor do C3.
- `utils.js` — utilitários compartilhados (`randomInt`).
- `main.js` — orquestração geral, input e gestão de andares.
- `Enemies.json` — dados externos de famílias de inimigos (stats, sprite, chance de spawn).
